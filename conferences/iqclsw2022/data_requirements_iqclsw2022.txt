# Structured manifest of the data files that the fetch (downloader) and process
# scripts in this directory rely on. make_app.py reads this file to decide
# whether a download can be skipped: when FORCE_DOWNLOAD is off, every file
# marked "required = yes" below must already exist in data/ for the download to
# be skipped; otherwise make_app runs the downloader.
#
# Format (simple, line-oriented; '#' starts a comment, blank lines ignored):
#   - Each file is a block introduced by a header line "[file: <pattern>]".
#   - <pattern> is a filename or a glob resolved relative to the data/
#     subdirectory of THIS conference folder.
#   - Inside a block, "key: value" lines give metadata. Recognized keys:
#       required     yes | no   (default yes) — whether the file must be present
#       produced_by  which script normally creates it (fetch / process / manual)
#       description  one-line human description
#       manual       how to obtain the file by hand if the downloader can't
#   - Continuation: a "manual" value may wrap onto following indented lines.

[file: Program_IQCLSW2022.pdf]

description: Full IQCLSW 2022 program PDF — cover page, day-by-day schedule
  (Tuesday 23 August through Sunday 28 August 2022) with chairs, times, talk
  titles and speakers, plus the numbered 1..32 poster catalog at the end. This

  (right-click the link and "Save link as…"); the IQCLSW 2022 program is
  archival, so this URL is expected to stay stable.

[file: TutorialSpeakers.html]
required: no
produced_by: fetch
description: Public "Keynote, Invited and Tutorial speakers" page HTML — a
  single HTML table with four columns (Name, Univ./Institute/Company, Type,
  Title) where the affiliation cell carries a country-flag emoji. This is the
  only source that ties each keynote/invited/tutorial talk to a FULL speaker
  name and an AFFILIATION (the PDF only prints initial+surname and no
  affiliations). The processor matches by surname against the PDF's talks to
  upgrade them in place. Optional: without it the keynote/invited/tutorial
  talks fall back to the PDF's terse "Initial. Surname" speaker text and have
  no institution.
manual: Open https://iqclsw.phys.ethz.ch/tutorial-speakers/ in a browser. Use
  "Save Page As…" and choose the "Webpage, HTML Only" option, saving into
  data/ as exactly TutorialSpeakers.html .
